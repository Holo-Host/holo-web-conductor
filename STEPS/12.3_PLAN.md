# Step 12.3: Test Audit - Fix Tautological Tests

**Status**: FUTURE WORK

## Overview

This step addresses tests that are internally self-consistent but don't actually test functionality that could break. These tests will always pass regardless of implementation bugs.

---

## Part 1: fishy Project Audit

### HIGH Severity Issues

#### 1. `packages/extension/src/lib/messaging.test.ts` (lines 131-160)
**Pattern**: Serialize-then-deserialize round-trip tests
**Problem**: Tests `serializeMessage(x)` → `deserializeMessage(y)` → equals `x`. If both functions have correlated bugs (e.g., both drop a field), test passes.
**Fix**: Compare serialized bytes against known msgpack binary format; test edge cases like null bytes, missing fields, malformed input.

#### 2. `packages/core/src/ribosome/serialization.test.ts` (lines 210-228)
**Pattern**: Round-trip self-verification
**Problem**: Same as messaging.test.ts - serialize/deserialize tested against each other.
**Fix**: Compare binary output against known msgpack format from Holochain; add test vectors.

#### 3. `packages/core/src/hash/hash.test.ts` (lines 39-60, 103)
**Pattern**: Derives expected values from the tested function itself
**Problem**: `holoHash.slice(35,39)` compared against `dhtLocationFrom32(hash32)` - both from same implementation path.
**Fix**: Add test vectors from actual Holochain hash outputs or `@holochain/client`.

## Part 2: hc-http-gw-fork Project Audit

### HIGH Severity Issues

#### 1. `src/routes/zome_call/tests/responses.rs` (15+ tests)
**Pattern**: Mock-only tests
**Problem**: Tests like `happy_zome_call`, `ribosome_errors_are_returned`, `app_not_found`, etc. only verify that mocks return what they were programmed to return. Expected error messages are hardcoded in both test and code.
**Fix**: Replace with integration tests using real `Conductor` instances.

#### 2. `src/auth/tests.rs` (lines 90-148)
**Pattern**: Mock keypair engineered to match
**Problem**: Tests construct signatures using the same key that validates them. `test_authorized_agent_with_valid_signature` creates `SigningKey::from_bytes(&[1u8; 32])` and signs with it - test data engineered to pass.
**Fix**: Add test vectors from ed25519 specification; test signature validation against real-world edge cases.

#### 3. `tests/websocket.rs` (lines 74-196)
**Pattern**: Logic duplication in setup
**Problem**: Test setup mirrors production code exactly. `connect_app_websocket` and `reuse_connection` tests construct data that obviously passes through code paths.
**Fix**: Test with unexpected inputs; verify behavior under stress conditions.

### MEDIUM Severity Issues

| File | Issue | Fix |
|------|-------|-----|
| `src/routes/zome_call/tests/validations.rs` | Expected values use same constants as code | Hardcode actual values in test; test edge cases |
| `tests/kitsune_integration.rs` | Overly broad error matching (accepts any error) | Assert specific error types |
| `tests/publish_integration.rs` | Conditional assertions (prints instead of failing) | Make assertions unconditional; use #[should_panic] |
| `tests/dht.rs` | Loose string containment assertions | Validate JSON structure with serde_json |

### LOW Severity Issues

- `src/auth/tests.rs` (lines 166-175): `test_auth_challenge_fields` tests struct field assignment (compiler behavior)
- **Fix**: Remove - these tests add zero value

## Implementation Plan

### Phase 1: Add Test Vectors (fishy)

**Goal**: Replace circular tests with known-good test vectors

1. **Hash test vectors** - `packages/core/src/hash/hash.test.ts`
   - Get actual hashes from Holochain/holochain-client for known inputs
   - Add tests comparing computed hashes to reference values

2. **Serialization test vectors** - `packages/core/src/ribosome/serialization.test.ts`
   - Capture msgpack bytes from Holochain for known payloads
   - Test serializer output matches reference bytes

3. **Genesis test vectors** - `packages/core/src/storage/genesis.test.ts`
   - Get genesis action sequence from real Holochain source chain
   - Verify action types and structure match

### Phase 2: Strengthen Assertions (hc-http-gw-fork)

**Goal**: Replace weak assertions with precise validation

1. **DHT tests** - `tests/dht.rs`
   - Replace string containment with JSON structure validation
   - Use `serde_json::from_str` to parse and validate fields

2. **Validation tests** - `src/routes/zome_call/tests/validations.rs`
   - Hardcode expected values instead of using shared constants
   - Add boundary tests (MAX_IDENTIFIER_CHARS - 1, MAX_IDENTIFIER_CHARS, MAX_IDENTIFIER_CHARS + 1)

### Phase 3: Replace Mock Tests with Integration Tests (hc-http-gw-fork)

**Goal**: Test real behavior, not mock consistency

1. **responses.rs** - Convert to integration tests
   - Use `SweetConductor` to create real conductor instance
   - Test actual error handling paths

2. **auth tests** - Add real signature test vectors
   - Get ed25519 test vectors from IETF RFC 8032
   - Test signature validation against known-good signatures

### Phase 4: Remove or Refactor Low-Value Tests

**Goal**: Clean up tests that test compiler behavior

1. **fishy shared/index.test.ts** - Either remove trivial `ok(42)` tests or expand to test error handling
2. **hc-http-gw-fork auth field tests** - Remove struct field assignment tests
