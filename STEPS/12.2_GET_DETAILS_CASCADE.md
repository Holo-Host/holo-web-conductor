# Step 12.2: Add Cascade Support to get_details

**Status**: PLANNED
**Priority**: High (functional gap)
**Goal**: Enable `get_details` host function to fetch details from the network when not found locally, matching the pattern used by `get` and `get_links`.

---

## Problem Statement

The `get_details` host function currently only queries local SQLite storage. Unlike `get` (which uses Cascade) and `get_links` (which always fetches network), `get_details` cannot find details about records created by other agents.

**Current Implementation** (`get_details.ts`):
```typescript
// Only local storage calls - no network fallback
storage.getEntryDetails(entryHash, dnaHash, agentPubKey);
storage.getAction(actionHash);
storage.getDetails(entryHashToQuery, dnaHash, agentPubKey);
```

**Expected Implementation** (like `get.ts`):
```typescript
const cascade = new Cascade(storage, getNetworkCache(), getNetworkService());
const details = cascade.fetchDetails(dnaHash, any_dht_hash);
```

---

## Impact

This gap affects hApps that use `get_details` to:
- Follow update chains (e.g., profiles zome's `get_latest()`)
- Check for deletes on records
- Display record history/provenance

Without cascade support, browser agents can only see details for records they created locally, not records created by other agents on the network.

## Implementation Plan

### Phase 1: Add fetchDetails to Cascade

**File**: `packages/core/src/network/cascade.ts`

Add new method:
```typescript
/**
 * Fetch details by hash using cascade pattern
 *
 * Order:
 * 1. Local storage
 * 2. Network cache (if implemented)
 * 3. Network (if enabled and available)
 */
fetchDetails(
  dnaHash: DnaHash,
  agentPubKey: Uint8Array,
  hash: AnyDhtHash,
  options?: CascadeOptions
): Details | null {
  const opts = { ...this.options, ...options };

  // Detect hash type
  const hashType = getHashType(hash);

  // 1. Try local storage
  if (hashType === HoloHashType.Entry) {
    const localDetails = this.storage.getEntryDetails(hash, dnaHash, agentPubKey);
    if (localDetails) {
      return { type: "Entry", content: this.formatEntryDetails(localDetails) };
    }
  } else {
    // Action hash - get action, then entry details
    const action = this.storage.getAction(hash);
    if (action && 'entryHash' in action) {
      const details = this.storage.getDetails(action.entryHash, dnaHash, agentPubKey);
      if (details) {
        return { type: "Record", content: this.formatRecordDetails(action, details) };
      }
    }
  }

  // 2. Try network cache (TODO: add cacheDetailsSync if needed)

  // 3. Try network
  if (opts.useNetwork && this.network?.isAvailable()) {
    const networkDetails = this.network.getDetailsSync?.(dnaHash, hash);
    if (networkDetails) {
      // Normalize byte arrays from JSON
      return normalizeByteArraysFromJson(networkDetails);
    }
  }

  return null;
}
```

### Phase 2: Update get_details Host Function

```typescript
import { Cascade, getNetworkCache, getNetworkService } from "../../network";

export const getDetails: HostFunctionImpl = (context, inputPtr, inputLen) => {
  const { callContext, instance } = context;
  const storage = getStorageProvider();
  const [dnaHash, agentPubKey] = callContext.cellId;

  const inputs = deserializeFromWasm(instance, inputPtr, inputLen);
  const input = inputs[0];
  const inputHash = input.any_dht_hash;

  // Create cascade for this lookup
  const cascade = new Cascade(storage, getNetworkCache(), getNetworkService());

  // Try cascade: local → cache → network
  const details = cascade.fetchDetails(dnaHash, agentPubKey, inputHash);

  if (!details) {
    return serializeResult(instance, [null]);
  }

  return serializeResult(instance, [details]);
};
```

## Response Format

The gateway returns Details in adjacently-tagged enum format:

**For Entry Hash**:
```json
{
  "type": "Entry",
  "content": {
    "entry": { "entry_type": "App", "entry": [...] },
    "actions": [...],
    "rejected_actions": [...],
    "deletes": [...],
    "updates": [...],
    "entry_dht_status": "Live"
  }
}
```

**For Action Hash**:
```json
{
  "type": "Record",
  "content": {
    "record": {
      "signed_action": { "hashed": {...}, "signature": [...] },
      "entry": { "Present": {...} }
    },
    "validation_status": "Valid",
    "deletes": [...],
    "updates": [...]
  }
}
```

The existing `get_details.ts` formatting logic should be reusable, but may need adaptation to handle network response format vs local storage format.
