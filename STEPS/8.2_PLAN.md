# Step 8.2: Op Signing Protocol

## Goal

Sign DhtOps with agent's private key for DHT validation. Browser extension signs ops locally, keeping the private key secure.

## Signing Flow

1. Generate DhtOps from committed action (Step 8.1)
2. Serialize each op for signing
3. Use Lair client to sign with agent's Ed25519 key
4. Wrap signed ops for transmission to gateway

## Signed Op Structure

Matches Holochain's `SignedDhtOp`:

```typescript
interface SignedDhtOp {
  signature: Uint8Array;  // 64-byte Ed25519 signature
  op: DhtOp;              // The operation being signed
}
```

## What Gets Signed

The signature covers the serialized `DhtOp` content, specifically:
- The op type
- The action data
- The entry data (if applicable)
