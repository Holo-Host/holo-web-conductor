# Step 8.3: Gateway Publish Endpoint

## Goal

Add HTTP endpoint to the gateway to accept published ops from browser extension and forward them to the DHT via kitsune2.

## Endpoint Design

**POST `/dna/:dna_hash/publish`**

Request body:
```json
{
  "ops": [
    {
      "signature": "<base64>",
      "op": {
        "type": "StoreRecord",
        "action": { ... },
        "entry": { ... }
      }
    }
  ],
  "author": "<base64 AgentPubKey>"
}
```

Response:
```json
{
  "success": true,
  "published_count": 3,
  "op_hashes": ["<base64>", ...]
}
```

## Processing Flow

1. Receive POST request with signed ops
2. Deserialize and validate each op
3. Verify signatures match claimed author
4. Call `OpStore.process_incoming_ops()` to store locally
5. Trigger kitsune2 publish to DHT authorities
6. Return success/failure response
