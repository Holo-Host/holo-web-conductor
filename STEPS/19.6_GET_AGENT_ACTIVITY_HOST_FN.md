# Step 19.6: Implement get_agent_activity Host Function

## Status: Open — Non-blocking for e2e test, needed for profiles zome

## Repo: fishy-step19

## Symptom

The `get_agent_activity` host function is a stub that returns `[]` (empty array). The profiles zome's `get_joining_timestamp_for_agent` calls `get_agent_activity` to find the earliest action timestamp for an agent. Because the stub returns `[]`, deserialization fails:

```
WASM error: get_joining_timestamp_for_agent received [129, 162, 79, 107, 144]
  → decodes to {Ok: []}
  → expected AgentActivity struct
```

The deserialization error is caught and logged but does not block profile creation or mew functionality.

## Current Implementation

**File**: [stubs.ts:44](packages/core/src/ribosome/host-fn/stubs.ts#L44)

```typescript
export const get_agent_activity = createEmptyArrayStub("get_agent_activity");
```

This returns `{Ok: []}` which cannot be deserialized as `AgentActivity`.

## Required Implementation

### 1. Understand the expected return type

`get_agent_activity` should return `AgentActivity`:

```rust
// From holochain/crates/holochain_zome_types/src/query.rs
pub struct AgentActivity {
    pub valid_activity: Vec<(u32, ActionHash)>,  // (seq_num, action_hash) pairs
    pub rejected_activity: Vec<(u32, ActionHash)>,
    pub status: ChainStatus,
    pub highest_observed: Option<HighestObserved>,
}

pub enum ChainStatus {
    Valid(ChainHead),
    Forked(ChainFork),
    Invalid(ChainHead),
}
```

### 2. Implementation approach: Local chain query

For a zero-arc node, `get_agent_activity` for the LOCAL agent can be answered from the local source chain (SQLite storage). For REMOTE agents, it would need a network query (new hc-membrane endpoint, similar to Step 19.2's `count_links` pattern).

**Minimum viable implementation** (local agent only):

```typescript
// packages/core/src/ribosome/host-fn/get_agent_activity.ts

export function get_agent_activity(
  storage: FishyStorage,
  dnaHash: DnaHash,
  agentPubKey: AgentPubKey,
  input: GetAgentActivityInput
): AgentActivity {
  // Query local source chain for actions by this agent
  const actions = storage.getSourceChain(dnaHash, input.agent_pubkey);

  // Build valid_activity as [(seq_num, action_hash)] pairs
  const valid_activity = actions.map(a => [a.seq, a.hash]);

  return {
    valid_activity,
    rejected_activity: [],
    status: { type: "Valid", content: { /* chain head */ } },
    highest_observed: null,
  };
}
```

### 3. Network query for remote agents (future)

For remote agents, would need:
1. New hc-membrane endpoint: `GET /dht/{dna}/agent_activity/{agent}`
2. Wire protocol: `GetAgentActivityReq` / `GetAgentActivityRes`
3. Cascade support in fishy (same pattern as `get_details`)

This can be deferred — the local-only implementation unblocks the profiles zome for the current agent.

### 4. Registration

Replace the stub in [stubs.ts](packages/core/src/ribosome/host-fn/stubs.ts) with an import of the new implementation. Register it in the host function dispatcher (same pattern as other host functions).
