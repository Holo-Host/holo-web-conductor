## Signal Flow Architecture

```
Browser Extension (WASM)
    │
    │ hdk::send_remote_signal(RemoteSignal { agents, signal })
    ▼
Host Function: send_remote_signal.ts
    │ - Deserialize RemoteSignal from WASM
    │ - For each target agent:
    │   - Build ZomeCallParams (provenance, cell_id, zome, fn, payload, nonce, expires_at)
    │   - Serialize to msgpack
    │   - Hash with blake2b-256
    │   - Sign with Lair client
    │ - Queue in callContext.remoteSignals
    ▼
Ribosome Worker
    │ - Return remoteSignals in zome call result
    ▼
Offscreen Document
    │ - Forward via WebSocket: { type: "send_remote_signal", dna_hash, signals }
    ▼
Gateway WebSocket Handler (websocket.rs)
    │ - Parse ClientMessage::SendRemoteSignal
    │ - Call gateway_kitsune.send_remote_signals()
    ▼
GatewayKitsune::send_remote_signals (kitsune_proxy.rs)
    │ - Get space for DNA
    │ - For each target agent:
    │   - Look up peer URL from peer_store
    │   - Create WireMessage::remote_signal_evt
    │   - Send via space.send_notify()
    ▼
Kitsune2 Network → Target Conductor/Browser
```

## Key Implementation Details

### ZomeCallParams Structure
Matches Holochain's `holochain_zome_types::zome_io::ZomeCallParams`:
```typescript
{
  provenance: AgentPubKey,      // Sender's public key
  cell_id: [DnaHash, AgentPubKey], // Target cell
  zome_name: string,           // Always "recv_remote_signal"
  fn_name: string,             // Callback function
  cap_secret: null,            // No capability needed
  payload: ExternIO,           // Signal data
  nonce: Uint8Array,           // 32-byte random nonce
  expires_at: number,          // Timestamp (microseconds)
}
```

### Signature Contract
- Hash: blake2b-256 of msgpack-serialized ZomeCallParams
- Sign: Ed25519 via Lair client (synchronous)
- Signature: 64 bytes

### Fire-and-Forget Semantics
- Host function returns immediately (null)
- Network delivery is asynchronous
- Failures logged but not reported to caller (matches Holochain behavior)

## Known Limitations

1. **No delivery confirmation** - Signals are fire-and-forget per Holochain semantics
2. **Peer discovery required** - Target agent must be in peer_store (discovered via kitsune2)
3. **Gateway kitsune2 must be enabled** - Requires `HC_GW_KITSUNE2_*` env vars
