# Step 12.2: DHT Publishing Debug Panel (Per-hApp)

## Goal

Add debug functionality to each installed hApp card in the popup:
1. Shows publish status for ops (pending, in-flight, failed) for that hApp's DNAs
2. Provides a "Republish All" button to regenerate and republish all ops from local chain data for all DNAs in that hApp

## UI Approach

**Per-hApp Debug Section** (not a separate tab):
- Add a "Debug" button on each hApp card in `happs.ts`
- Clicking it expands a debug section showing:
  - Status counts (pending/in-flight/failed) aggregated across all DNAs in the hApp
  - Per-DNA breakdown if multiple DNAs
  - "Retry Failed" button
  - "Republish All" button

## Current Architecture

**Existing Infrastructure:**
- `PublishTracker` (packages/core/src/dht/publish-tracker.ts) - IndexedDB persistence for pending publishes with status tracking
- `PublishService` (packages/core/src/dht/publish-service.ts) - Orchestrates publish workflow with batching and retries
- `PendingPublish` records have: `id`, `op`, `basis`, `status` (Pending/InFlight/Published/Failed), `retryCount`, `lastAttempt`, `error`
- `happs.ts` - Shows installed hApps as cards with Enable/Disable/Uninstall buttons

**Key Insight:**
- Once ops are successfully published, they're removed from the tracker
- So the panel would show: currently pending/in-flight + failed (not historical published)
- For "republish all", we need to regenerate ops from SQLite storage records

## Implementation Plan

### Phase 1: Backend Message Handlers

**File: `packages/extension/src/lib/messaging.ts`**
Add new message types:
```typescript
PUBLISH_GET_STATUS = "publish_get_status",      // Get counts for a hApp context
PUBLISH_RETRY_FAILED = "publish_retry_failed",  // Retry all failed ops for a hApp
PUBLISH_ALL_RECORDS = "publish_all_records",    // Regenerate + publish all ops for a hApp
```

**File: `packages/extension/src/background/index.ts`**
Add handlers:
1. `handlePublishGetStatus(contextId)` - Return {pending, inFlight, failed} counts for all DNAs in hApp
2. `handlePublishRetryFailed(contextId)` - Reset all Failed → Pending for all DNAs in hApp
3. `handlePublishAllRecords(contextId)` - For each cell in hApp, iterate all records and queue ops

### Phase 2: PublishTracker Enhancements

**File: `packages/core/src/dht/publish-tracker.ts`**
Add methods if not present:

- `getAllPendingForDnas(dnaHashes: DnaHash[])` - Return all non-published ops for display
- `resetFailedForDnas(dnaHashes: DnaHash[])` - Reset all failed to pending status

### Phase 4: Republish All Implementation

**Data Flow:**
```
Popup (happs.ts)
  → [PUBLISH_ALL_RECORDS] → Background (index.ts)
    → [GET_ALL_RECORDS] → Offscreen (index.ts)
      → [GET_ALL_RECORDS] → Ribosome Worker
        → SQLite query
      ← records[]
    ← records[]
  ← produces ops → queues for publish
```

**Implementation Steps:**

1. **Ribosome Worker** (`packages/extension/src/offscreen/ribosome-worker.ts`)
   - Add new message handler for `GET_ALL_RECORDS`
   - Queries all actions for a cell using existing `queryActions()`
   - For each action with entryHash, also gets the entry
   - Returns array of records: `{ action, entry }[]`

2. **Offscreen Document** (`packages/extension/src/offscreen/index.ts`)
   - Add handler for `GET_ALL_RECORDS` message from background
   - Forwards to ribosome-worker, returns response

3. **Background Handler** (`packages/extension/src/background/index.ts`)
   - `handlePublishAllRecords(contextId)`:
     - Get HappContext by ID → access `context.dnas[]`
     - For each DnaContext, get dnaHash and agentPubKey
     - Send `GET_ALL_RECORDS` to offscreen for each cell
     - Convert returned records to Holochain Record format
     - Call `produceOpsFromRecord()` on each
     - Queue ops via PublishTracker (not PublishService - we just queue, not process)
   - Return: `{ cellsProcessed: number, opsQueued: number, errors: string[] }`

**Key Files:**
- `packages/extension/src/offscreen/ribosome-worker.ts` - Add GET_ALL_RECORDS handler
- `packages/extension/src/offscreen/index.ts` - Forward GET_ALL_RECORDS
- `packages/extension/src/background/index.ts` - handlePublishAllRecords
- `packages/core/src/dht/produce-ops.ts` - produceOpsFromRecord (existing)
