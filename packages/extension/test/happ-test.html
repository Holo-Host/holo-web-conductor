HWC Client Library Test

Tests the @hwc/client WebConductorAppClient against the bundled test zome. Uses the standard @holochain/client callZome shape, not the direct window.holochain.* API. No running linker needed -- local operations work fine with the linker status showing as disconnected.

1. Extension Detection

2. Authorization

3. Install test.happ

Loads test.happ from the test directory and installs it via window.holochain.installHapp().

4. Client Connection

Connects via WebConductorAppClient.connect(). A default linker URL is configured under the hood; the linker does not need to be running for local operations.

5. Zome Call Tests via Client

Each button calls a zome function through appClient.callZome() using the standard { cell_id, zome_name, fn_name, payload, provenance, cap_secret } shape.

6. Signal Test via Client

Subscribes via appClient.on('signal', callback) then emits a signal through the zome.

7. Connection Status disconnected

Tracks both the extension connection and linker health via appClient.onConnection('connection:change', callback). The badge reflects overall status (connected = extension link is up). HTTP/WS detail shows linker health -- "down" and "fetch failed" are normal when no linker is running; local zome operations still work.
